Checkpoint Preparation

Before the group attends Checkpoint 1 meeting, please carry out the following steps:

Contact your team members and agree on a shared communication channel, e.g. Teams, Discord etc. If consensus cannot be reached, then please use Teams by default as everyone has access to it.
Create your own GitHub account if you don't already have one (see Exercises in Module 1 on the LMS front page).
Have one person of your team create an empty GitHub repository and add everyone as collaborators to that repository.
Have everyone clone the empty repository to their laptop computer and test that they can push and pull from it. Don't worry about making a mess!
Have a brief brainstorming session about what sort of web application you would like to build!
The Checkpoint Meeting

All members of the team must bring their laptop to the meeting with a local copy of the repository on (see Preparation instructions above). During the 20 minute meeting:

everyone will introduce themselves.
the facilitator will ask you to demonstrate that each of you understand how to commit, push, pull, open PRs, perform reviews and resolve merge conflicts.
the facilitator will make sure that your GitHub project repo settings are configured correctly and will advise you how to reset the repository to a blank state.
they will ask you about any ideas you have for your project.
If a member of the group is unable to demonstrate that they can use Git/GitHub, then the facilitator will ask that person to come back to future sessions and repeat the exercise. Repeated failures to demonstrate these skills will result in deductions to the Agile Methodology mark.
